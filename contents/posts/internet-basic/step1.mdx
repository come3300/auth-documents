## Phase1 インターネットの仕組み

---

![Photo by Minh Pham](/images/documents/internet-basic/network.png)

### このセクションの目標

インターネットがどのようにして動き、
ネットに表示してくれているのかを理解しよう

### なぜ必要なのか

なぜ、インターネットが動くのか理解しなくてはいけないのか,説明します。
結論インターネットが動く仕組みを理解しなくてもプログラミングを
用いてコードを書くだけなら可能です。ですが、Webが表示されるまででは
普段私たちが使っているインターネットがどのような仕組みで
動いているのかを大まかに理解していきましょう。
※初めはざっくりで大丈夫です。

---

![Photo by Minh Pham](/images/documents/internet-basic/watch-youtube.png)

皆さんが普段使っているYouTubeがどのように表示されるのかを例に
解説していきます。

#### 1,URLに見たい動画のURLをうつ

まずはじめに皆さんがYouTubeで動画をみたいと思った動画のURLを入力するはずです。
では内部でそのようなことが起きているのかというと

![Photo by Minh Pham](/images/documents/internet-basic/リクエストレスポンス.png)

入力したURLを元にこの動画を取ってきてね!と
私たちのPCがサーバーに情報を要求してくれます。

これをクライアントがサーバーにリクエストしたといいます。

クライアント→検索サイトで検索したユーザー
リクエスト→サーバーに情報を要求すること

#### 2,サーバーがリクエストした情報を返してくれる

次にリクエストした情報をサーバーがクライアント（検索したユーザー）に
返してくれます。

![Photo by Minh Pham](/images/documents/internet-basic/httpについて.png)

このサーバーが要求した情報を返してくれることを
**レスポンス**と言います。

普段皆さんがLINEやチャットで連絡しているときに返信が早い人に対して
レスが速いと言いませんか？

このレスとはレスポンスのことで
連絡したことに対して相手が連絡を返してくれることを指しますが
Webの世界でも同じようにサーバーがリクエストした情報を返してくれることを
レスポンスと言います。

### HTTPについて学んでいこう

ここまででどのようにWebブラウザが表示されているのか
大まかに理解できたと思います。

これからエンジニアを目指すみなさんは
もう少し踏み込んだことも学習していきましょう。

#### HTTPメソッドについて

![Photo by Minh Pham](/images/documents/internet-basic/http.png)

サーバから何か情報を引っ張ってきてWebブラウザ上に
表示するにはリクエストしなくてはいけないと学びましたね。
ではこれから、そのHTTPメソッドの基本的な4種類

・GET
・POST
・PUT
・DELETE

について学んでいきましょう。

#### こんな時に役に立つ

これから学ぶことがでは実際にどんな時に役に立つのかというと
主にDB操作やルーティングといった処理をしていくときにこの知識が役に立ちます。

```PHP
Route::get('articles/{id}', 'BasicController@showArticle');
```

上記はLaravelで書かれたコードの一部を抜粋したものになります。

Route::getとありますよね、これはgetメソッドを用いて対象の記事を
サーバーから取ってくる処理になるのですが今から学ぶ,HTTPメソッドを
理解していないとこういった処理を実装することが難しくなります。

### CRUDとは

![Photo by Minh Pham](/images/documents/internet-basic/CRUD.png)

| 名前 | 役割 | 具体例 |
|:-----------|:------------|:------------|
| Create | 作成 | 新規ユーザー登録、新しい投稿を作成 |
| Read | 読み込み | ユーザー情報の表示、記事一覧の閲覧 |
| Update | 更新 | プロフィール情報の編集、投稿の修正 |
| Delete | 削除 | アカウントの削除、投稿の削除 |

Create（作成）、Read（読み込み）、Update（更新）、Delete（削除）
の頭文字であり、データ操作の基本はこの4つと言われています。

<Note>
**身近な例：SNSアプリ**

SNSを使うときの操作をCRUDに当てはめてみましょう。

- **Create（作成）**：新しい投稿を作成する、友達を追加する
- **Read（読み込み）**：タイムラインを見る、他の人のプロフィールを見る
- **Update（更新）**：投稿を編集する、プロフィール写真を変更する
- **Delete（削除）**：投稿を削除する、友達を削除する

普段何気なく使っているアプリの裏側では、これらのCRUD操作が行われているのです。
</Note>

※エンジニアとして実務をこなしていくとちょこちょこ目にする機会があるかもしれません

HTTPメソッドはこのCRUDに対して、
下記のように対応しています

| 名前   | 役割   |
| :----- | :----- |
| Create | POST   |
| Read   | READ   |
| Update | UPDATE |
| Delete | DELETE |

### GETメソッド

![Photo by Minh Pham](/images/documents/internet-basic/CRUD-GET.png)

GETメソッドはリソースを取得するためのメソッドです。

Webページを閲覧したいときは、
「Webページを取得したいです」というGETリクエストをクライアントが送信し、
Webページのhtmlなどを受け取ります。

例

> ・ある記事を取得したい
> ・特定のユーザーの情報を取得したい

以下のURLにGETリクエストを送信した場合、
ユーザー一覧を取得するという意味になります。

"https://example.com/user"

また、GETでは大抵リクエストパラメータによって
リソースの取得方法をコントロールすることもできます。

例えば、

"https://example.com/user"
はユーザー一覧を取得できる一方で

"https://example.com/user?id=1"
といった具合でURLにパラメータを付与すると、
ユーザーIDが「1」のユーザーのみを取得することができます。

<Note>
**日常生活の例：YouTubeで動画を見る**

YouTubeで動画を見るとき、あなたは次のことをしています：

1. 検索窓に「猫 かわいい」と入力
2. 動画のサムネイルをクリック
3. 動画が再生される

この一連の流れは全て**GETリクエスト**です。

- 検索結果を「取得」
- 動画データを「取得」
- コメント一覧を「取得」

見ているだけ、読んでいるだけの操作は基本的にGETメソッドが使われています。
サーバーのデータは変更せず、ただ情報を「もらう」だけの操作です。
</Note>

### POSTメソッド

POSTメソッドはリソースを送信するためのメソッドです。

皆さんが何かのお問合せフォームで質問をして送信ボタンを
押したとします。するとその送信した情報をサーバーに送ってくれます。

![Photo by Minh Pham](/images/documents/internet-basic/CRUD-POST.png)

よくあるお問合せフォームには送信したら送信完了の連絡が来ると思います。
この完了の連絡をレスポンスして返してくれてます。

例
・入力フォームに記入した情報の送信
・添付したファイルを送る
・画像を送信する

```
<form action="sample.php" method="post">
  <input type="textarea" name="confirm" value="">
  <input type="submit" value="送信">
</form>
```

これから学習するhtmlでは上記のように書くことで
情報をpostメソッドとして送ってくれています。

<Note>
**日常生活の例：TwitterやInstagramに投稿する**

SNSで新しい投稿をするとき、POSTメソッドが使われています。

例：Twitterに投稿する場合
1. 「いま何してる？」の欄にテキストを入力
2. 写真を添付する
3. 「ツイート」ボタンを押す
4. → サーバーに新しいツイートデータが**作成**される

POSTの特徴：
- サーバーに新しいデータを**追加**する
- 同じ内容を2回送信すると、2つの投稿が作られる（重複する）
- フォーム送信やファイルアップロードで使われる

GETが「見るだけ」なのに対し、POSTは「新しく作る」操作です。
</Note>

### PUTメソッド

![Photo by Minh Pham](/images/documents/internet-basic/CRUD-PUT.png)

PUTはリソースの更新/作成を行うためのメソッドです。

リソースの作成はほとんどPOSTが使われることが多いため、
ほぼ更新用のメソッドという認識で問題ないかなと思います。

> 既存ユーザー情報の更新
> 既存ブログ記事の更新
> 以前に投稿したレビューの更新

といった場面で利用されます。

<Note>
**日常生活の例：プロフィール情報を編集する**

SNSやショッピングサイトでプロフィールを編集するとき、PUTメソッドが使われます。

例：Amazonで配送先住所を変更する場合
1. アカウント設定を開く
2. 住所情報を編集する
3. 「保存」ボタンを押す
4. → 既存の住所データが**更新**される

PUTの特徴：
- **既にあるデータ**を書き換える
- 新規作成ではなく、上書き更新
- 「編集」「変更」の操作で使われる

POSTが「新規作成」なのに対し、PUTは「既存のものを更新」する操作です。
</Note>

### DELETEメソッド

![Photo by Minh Pham](/images/documents/internet-basic/CRUD-DELETE.png)

DELTEはリソースの削除を行うためのメソッドです。
主にDBの情報を削除するときに使います。

例

> ・ユーザーの削除
> ・ブログ記事の削除
> ・ツイートの削除

<Note>
**日常生活の例：投稿を削除する**

SNSで投稿を削除するとき、DELETEメソッドが使われています。

例：Instagramの投稿を削除する場合
1. 削除したい投稿を開く
2. メニューから「削除」を選択
3. 確認ダイアログで「削除」をタップ
4. → サーバーからその投稿データが**削除**される

DELETEの特徴：
- データを完全に削除する
- 削除したら基本的には元に戻せない
- 「削除」「退会」の操作で使われる

慎重に扱うべきメソッドなので、実装時には確認ダイアログを
表示することが一般的です。
</Note>

## まとめ：HTTPメソッドとCRUDの対応

ここまで学んだHTTPメソッドとCRUDの関係を整理しましょう。

### 対応表

| CRUD | HTTPメソッド | 操作 | 日常の例 |
|:-----|:-------------|:-----|:---------|
| **C**reate | POST | 新規作成 | SNSに新しい投稿をする |
| **R**ead | GET | 読み込み | YouTubeで動画を見る |
| **U**pdate | PUT | 更新 | プロフィール情報を編集する |
| **D**elete | DELETE | 削除 | 投稿を削除する |

<InfoBox>
**覚え方のコツ**

HTTPメソッドは英単語の意味そのままです。

- **GET**：「取ってくる」→ データを取得
- **POST**：「投稿する」→ 新しいデータを送信
- **PUT**：「置く」→ データを上書き
- **DELETE**：「削除する」→ データを削除

**実務でよく使う順番：**
1. **GET**と**POST**：この2つで9割をカバー
2. **PUT**と**DELETE**：RESTful APIでよく使う

初心者の方は、まずGETとPOSTの違いをしっかり理解することから始めましょう。

**注意点：**
- GETは何度実行してもサーバーのデータは変わらない（安全）
- POST/PUT/DELETEはサーバーのデータを変更する（慎重に扱う）

</InfoBox>

### 実際のコード例

```php
// Laravel のルーティング例
Route::get('/articles', 'ArticleController@index');      // 記事一覧を取得
Route::post('/articles', 'ArticleController@store');     // 新しい記事を作成
Route::put('/articles/{id}', 'ArticleController@update'); // 記事を更新
Route::delete('/articles/{id}', 'ArticleController@destroy'); // 記事を削除
```

このように、HTTPメソッドを適切に使い分けることで、
処理の内容が一目でわかるコードを書くことができます。

## 小ネタ

#### インターネットとWebの違い

インターネットとWebの違いについて皆さんはご存知でしょうか？

**_インターネット_**

> 世界中のコンピューターをつなぎ合わせる巨大なネットワーク

**_Web_**

> HTTPという仕組みを利用しホームページやブログを閲覧すること

つまりインターネットはホームページやブログを世界中のコンピューターにつなぎ合わせる役割があり、
WebはそのHTTPの仕組みを利用して特定のブログやホームページを閲覧できるようにすることになります。
