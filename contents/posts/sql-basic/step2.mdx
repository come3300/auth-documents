## なぜ変数が必要なのか？
{/* TODO イメージ画像 */}
プログラミング言語を学んでいく上でこの「変数」という概念は必ず理解しなければならないものです。
実際にアプリやシステムを開発していく上でこの「変数」を使わないなんてことは絶対にあり得ません。

### 変数がない世界を想像してみよう
まず、変数がないプログラミングを想像してみてください:
javascript// 変数を使わない場合
```JavaScript
console.log("こんにちは、田中さん!");
console.log("田中さんの年齢は25歳です");
console.log("田中さんのメールアドレスはtanaka@example.comです");
```
もし「田中さん」を「佐藤さん」に変更したくなったら...すべての行を書き換える必要があります。たった3行でも大変なのに、100行、1000行あったらどうでしょう?
変数があると何が変わる?
変数を使えば、こうなります:
javascript// 変数を使う場合
```JavaScript
let name = "田中さん";
let age = 25;
let email = "tanaka@example.com";

console.log("こんにちは、" + name + "!");
console.log(name + "の年齢は" + age + "歳です");
console.log(name + "のメールアドレスは" + email + "です");
```
名前を変更したい時は、1箇所だけ変更すればOKです!
javascriptlet name = "佐藤さん"; // ここだけ変更
変数が必要な5つの理由
#### 1. データを保存する「箱」として
変数は「データを入れておく箱」のようなものです。
javascriptlet score = 100; // スコアという箱に100を保存
score = score + 10; // 箱の中身を取り出して10を足し、また箱に戻す
console.log(score); // 110
箱がないと、計算結果をどこにも保存できません。
#### 2. 同じ値を何度も使う

```JavaScript
javascriptlet tax = 0.1; // 消費税率

let price1 = 1000;
let totalPrice1 = price1 + (price1 * tax);

let price2 = 2500;
let totalPrice2 = price2 + (price2 * tax);

let price3 = 500;
let totalPrice3 = price3 + (price3 * tax);
```
もし消費税率が変わったら、let tax = 0.1;をlet tax = 0.08;に変更するだけで、
すべての計算が自動的に更新されます!

#### 3. わかりやすいコードを書く
javascript// わかりにくい
```JavaScript
let a = 500 * 0.1;
```

// わかりやすい
```JavaScript
let itemPrice = 500;
let discountRate = 0.1;
let discount = itemPrice * discountRate;
```
変数名を見れば、何のデータかすぐわかります。
#### 4. 計算結果を保存する
javascript// ユーザーの入力を受け取る
```JavaScript
let number1 = 10;
let number2 = 20;

// 計算結果を変数に保存
let sum = number1 + number2;
let average = sum / 2;

console.log("合計: " + sum);      // 30
console.log("平均: " + average); // 15
```
計算結果を変数に保存しないと、後で使えません。
#### 5. データが変化するプログラムを作る
```JavaScript
javascriptlet lives = 3; // ゲームの残機

console.log("残機: " + lives); // 3

// 敵に当たった!
lives = lives - 1;
console.log("残機: " + lives); // 2

// また敵に当たった!
lives = lives - 1;
console.log("残機: " + lives); // 1
```
変数があるから、プログラムが「記憶」を持てて、状態を変化させられるんです。
